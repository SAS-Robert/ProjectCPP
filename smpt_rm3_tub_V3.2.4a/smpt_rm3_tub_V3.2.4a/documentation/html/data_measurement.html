Data-measurement

Dm – Send commands

The header smpt_dm_client.h needs to be included to use the data measurement (dm) functions.
Each command is send using the smpt_send_dm_*() functions. Depending on the command, a struct or the packet number is used as a parameter. To check if a struct is valid, the functions in smpt_dm_packet_validity.h (smpt_is_valid_dm_*()) can be used.

  • smpt_send_dm_init():
    • initializes the emg chip.
    • can be used to configure the measurement and the channels.
    • can be used to read / write the registers of the internal EMG chip.
    • the registers are set to measure EMG with both channels at 4 kHz.
  • smpt_send_dm_start():
    • starts measurement. Smpt_dm_data is sent periodically from the device.
  • smpt_send_dm_stop():
    • stops the measurement.
    • to restart the measurement, the measurement needs to be initialized again.

Dm – Receive responses

Every command that has been sent generates a response (ack). This response contains the same packet number as the sent command and a result variable, which provides execution and error information.

  • smpt_new_packet_received():
    • can be used to check if a new packet has been received.
    • triggers the read of the serial device.
  • smpt_last_ack():
    • returns general information about the response.

Some commands have additional parameters. Use the following functions to get them:

  • smpt_get_dm_init_ack():
    • returns the parameters of Smpt_dm_int_ack (response to dm_init).

The measured data received can be unpacked using

  • smpt_get_dm_data()
